# Support

Thanks for using MediaForge! Here is how to get help.

## Before Asking

1. Run the built-in diagnostics:
   ```bash
   mediaforge doctor
   ```
   It reports missing tools (FFmpeg, curl_cffi, mutagen, a JS runtime, …) and
   how to install them — most issues are environment-related.
2. Check the [README](README.md), especially the **Troubleshooting** and **FAQ**
   sections.
3. Search [existing issues](https://github.com/your-username/MediaForge/issues)
   and [discussions](https://github.com/your-username/MediaForge/discussions).

## How to Get Help

| I want to… | Use |
|------------|-----|
| Ask a question or share an idea | [GitHub Discussions](https://github.com/your-username/MediaForge/discussions) |
| Report a reproducible bug | [Bug report issue](https://github.com/your-username/MediaForge/issues/new/choose) |
| Request a feature | [Feature request issue](https://github.com/your-username/MediaForge/issues/new/choose) |
| Report a security issue | [SECURITY.md](SECURITY.md) (do not open a public issue) |

## Writing a Good Report

Include:

- MediaForge version (`mediaforge --version`)
- Output of `mediaforge doctor`
- Your OS and Python version
- The exact command / menu choices you made
- The full error message (redact anything private)

## Recommended Discussion Categories

If you maintain a fork, enable GitHub Discussions with these categories:

- **Announcements** — releases and roadmap updates (maintainers only)
- **Q&A** — usage questions with accepted answers
- **Ideas** — feature proposals before they become issues
- **Show and tell** — share what you built with MediaForge
